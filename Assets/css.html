Cascading Style Sheet

Keuntungan ketika menerapkan CSS

  1. Dapat mengontrol dan menerapkan layout (tata letak) secara presisi
  2. Menghindari pekerjaan yang berulang-ulang dalam menerapkan styling
  3. Didukung banyak browser

Cara stylesheet bekerja

  1. Dimulai dari sebuah dokumen yang sudah ditandai dengan tag elemen HTML.
  2. Menuliskan aturan styling untuk menentukan tampilan elemen HTML.
  3. Melampirkan file .css pada HTML.

Menulis Aturan Styling

Terdapat 2 bagian pada sebuah rule. Bagian pertama adalah identitas elemen (selector) dan kedua adalah deklarasi atau instruksi yang akan diterapkan pada sebuah selector.

    h1{
      color: green;    
    }
            
    p{
      font-size: small;
      font-family: sans-serif;
    }

Selector

Pada contoh di atas, <h1> dan <7> digunakan sebagai selector. Properti dan nilainya yang terdapat dalam declaration/declaration block akan diterapkan pada seluruh elemen <h1> dan <p> yang ada pada dokumen HTML.

Declaration

Deklarasi terdiri dari pasangan properti dengan nilainya. Satu rule bisa diberi lebih dari satu deklarasi seperti contoh pada selector p di atas. Setiap deklarasi harus diakhiri dengan semicolon (;) sebagai tanda diakhirinya sebuah deklarasi.
CSS tidak memperhatikan spasi, sama seperti bahasa pemrograman pada umumnya. Walaupun seperti itu, untuk satuan nilai semisal px, em, dan lainnya harus dituliskan tanpa spasi pada nilainya. Contohnya seperti dibawah: 

            h1{
                margin: 2em;
            }

Komentar Pada CSS

Komentar digunakan untuk menjelaskan kode dan dapat membantu ketika mengedit kode di lain waktu. Hal yang ditulis tidak akan ditampilkan di browser namun bisa dilihat pada text editor.
Cara penulisan comment pada CSS adalah sebagai berikut:
/* */

Melampirkan Styling Pada Dokumen HTML

  • External Style Sheet

    • External Style Sheet adalah berkas terpisah yang di dalamnya hanya ada CSS rules. Berkas harus berekstensi .css dan dihubungkan dengan dokumen HTML. Dengan cara ini, satu berkas styling dapaat digunakan oleh banyak berkas HTML.

    Cara Menghubungkan berkas .css dengan dokumen HTML 
        <head>
        <meta charset="8-UTF">
        <title>Styling</title>
        <link href="Style.css">
    </head>
    Catatan: Nama berkas yang mengandung "min.css" adalah penamaan format berkas CSS yang sudah diminimalkan dengan menghilangkan white space yang tidak digunakan.

  • Embedded Style Sheet

    • Embedded Style Sheet adalah kumpulan rules yang dituliskan dalam berkas HTML dengan menggunakan elemen <style>. Dengan demikian, CSS rules yang dituliskan hanya dapat dicakup oleh satu berkas HTML. Penulisan rules harus dituliskan dalam elemen <style>.

    Penulisan Rules 
        <head>
        <title>
        <style>
          h1{
            color: blue;
          }
        </style>
    </head>

  • Inline Style

    • Inline Style adalah stylingyang diterapkan secara langsung pada elemen HTML dengan menggunakan atribut style.
    Dalaam metode ini kita tidak memerlukan selector karena style hanya diterapkan pada elemen tempat atribut style diterapkan.

    Inline Style 
    <p> style="color: blue;"> paragraf </p>

CSS Conception

Inheritance

Dapat mewarisi properti style tertentu dari suatu elemen ke elemen-elemen di dalamnya (child-elements). Contohnya CSS rules yang ditetapkan untuk elemen <body> akan diterapkan pada seluruh child-elements secara otomatis.

Group Selector

Jika beberapa selector yang berbeda memiliki penerapan properti-properti yang sama, kita dapat menggabungkannya menggunakan group selector.

Contoh Penggunaan 
    h1{
        font-family: sans-serif;
    }
    h2{
        font-family: sans-serif;
    }
    /*Diubah menjadi*/
    h1, h2{
        font-family: sans-serif;
    }

Rule Order

cascading artinya "mengalir". Demikian halnya dengan alur kerja CSS dalam membaca kode, mengalir dari atas ke bawah. Oleh karena itu, urutan dalam penulisan rules harus diperhatikan, terutama saat terjadi konflik. Konflik dapat terjadi ketika menerapkan beberapa styling pada satu dokumen HTML dan menimpa styling yang telah diterapkan sebelumnya baik disadari maupun tidak.

HTML CSS 
<!DOCTYPE html>
<html>
    <head>
    <meta charset="UTF-8" />
    <title>Judul</title>

    <link rel="stylesheet" href="Style.css">

    <style>
        p{
         color: blue;
        }
    </style>
    </head>
    <body>
    <p>
      Lorem ipsum dolor sit amet.
    </p>
    </body>
</html>
                
  p{
   color: red;
  }
Dalam contoh diatas, warna yang akan diterapkan pada elemen <p> adalah warna biru karena merupakan warna yang paling akhir didefinisikan.

Namun, kita bisa membuat sebuah property styling agar dianggap penting oleh browser untuk diterapkan dan urutan tidak diperhatikan dengan menambahkan keyword !important pada akhir nilai propertinya.

HTML CSS 
<!DOCTYPE html>
<html>
    <head>
    <meta charset="UTF-8" />
    <title>Judul</title>

    <link rel="stylesheet" href="Style.css">

    <style>
        p{
         color: blue;
        }
    </style>
    </head>
    <body>
    <p>
      Lorem ipsum dolor sit amet.
    </p>
    </body>
</html>
                
  p{
   color: red !important;
  }
Dalam contoh diatas, warna yang akan diterapkan pada elemen <p> adalah warna merah karena karena terdapat keyword !important pada akhir nilai properti.

Selektor Dasar

Type Selector

Menggunakan nama elemen sebagai target untuk diterapkannya rule.

Class Selector

Menetapkan target elemen berdasarkan nilai dari atribut class yang diterapkan pada elemennya. Untuk penulisan selector-nya diawali dengan tanda titik (.) dan dilanjutkan dengan nama class nya. <!DOCTYPE html>

HTML CSS 
    <!DOCTYPE html>
    <html>
    <head>
       <meta charset="UTF-8"
       <title>Judul</title>
    </head>
    <body>
      <p class="baris1">dolor sed viverra</p>
      <p class="baris2">in nibh mauris</p>
      <p class="baris3">a diam sollicitudin</p>
    </body>
    </html>
                    
    .baris1{
        color: red;
    }
    
    .baris2{
        color: blue;
    }

    .baris3{
        background-color: green;
    }
Output (hasil) dari ketiga elemen <p> tersebut masing-masing adalah:
baris 1 merupakan teks berwarna merah
baris 2 merupakan teks berwarna biru
baris 3 merupakan teks dengan latar belakang berwarna hijau

Class bersifat sharable sehingga dapat diterapkan pada banyak elemen dengan tipe elemen yang berbeda-beda seperti pada contoh di bawah: 
    <h1 class="baris1">Heading dengan teks berwarna merah</h1>
    <p class="baris1">Paragraf dengan teks berwarna merah</p>
                    
    <h1 class="baris1">Heading dengan teks berwarna merah</h1>
    <p class="baris1 baris3">Paragraf dengan teks berwarna merah dan background berwarna hijau</p>
